1.
    编译时会报以下错误：
    $ gcc hello.c 
    hello.c:1:18: fatal error: gegl.h: No such file or directory
    compilation terminated.
    resolve:
    # yum provides */gegl.h
    gegl-devel-0.2.0-1.fc17.x86_64 : Headers for developing programs that will use
    : gegl
    Repo : fedora
    Matched from:
    Filename : /usr/include/gegl-0.2/gegl.h

    查出来的结果是 gegl-devel 包提供了该文件，安装该软件包:
    # yum install gegl-devel
2.
    安装好 gegl-devel 包后，需要在编译时通过 pkg-config 提供头文件和库:

    $ gcc hello.c `pkg-config --libs --cflags gegl`
    Package gegl was not found in the pkg-config search path.
    Perhaps you should add the directory containing `gegl.pc'
    to the PKG_CONFIG_PATH environment variable
    No package 'gegl' found
    新的错误是未找到 gegl.pc
    resolve:
    检查下 gegl-devel 提供的是什么：
    # rpm -ql gegl-devel | grep .pc
    /usr/lib64/pkgconfig/gegl-0.2.pc
    原来提供的是 gegl-0.2.pc
    然后在 pkg-config 里使用 gegl-0.2:
    $ gcc hello.c `pkg-config --libs --cflags gegl-0.2`
    /usr/bin/ld: /tmp/cc1I2skp.o: undefined reference to symbol 'g_thread_init'
    /usr/bin/ld: note: 'g_thread_init' is defined in DSO /lib64/libgthread-2.0.so.0 so try adding it to the linker command line
    /lib64/libgthread-2.0.so.0: could not read symbols: Invalid operation
    collect2: error: ld returned 1 exit status
3.
    新的错误提示没有找到提供了 g_thread_init 的库文件，同时提示 libgthread-2.0.so.0 含有 g_thread_init。
    先找库文件在那个包里:
    # yum provides /lib64/libgthread-2.0.so.0
    glib2-2.32.3-1.fc17.x86_64 : A library of handy utility functions
    Repo : @updates
    Matched from:
    Filename : /lib64/libgthread-2.0.so.0
    
    查出来的结果是在 glib2 包里，这就需要安装 glib2-devel 包，这个包我是安装了的，没有安装就安装:
    yum install glib2-devel
4.
    然后查看下 glib2-devel 提供的 pkg-config 文件：

    $ rpm -ql glib2-devel | grep .pc
    /usr/include/glib-2.0/gio/gtcpconnection.h
    /usr/lib64/pkgconfig/gio-2.0.pc
    /usr/lib64/pkgconfig/gio-unix-2.0.pc
    /usr/lib64/pkgconfig/glib-2.0.pc
    /usr/lib64/pkgconfig/gmodule-2.0.pc
    /usr/lib64/pkgconfig/gmodule-export-2.0.pc
    /usr/lib64/pkgconfig/gmodule-no-export-2.0.pc
    /usr/lib64/pkgconfig/gobject-2.0.pc
    /usr/lib64/pkgconfig/gthread-2.0.pc
    
    从需要的函数 g_thread_init 这个字面就能猜测到需要 gthread-2.0.pc
    然后在 pkg-config 里提供 gthread-2.0:
    $ gcc hello.c `pkg-config --libs --cflags gegl-0.2 gthread-2.0`
    没有报错，说明编辑成功，就可以运行编译处理的软件了
